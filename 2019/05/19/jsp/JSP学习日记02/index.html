    浏览器跳转的两种方式

    • 转发 发起一次请求 浏览器不知道变化 地址不改变
    • 重定向 发起两次请求 浏览器知道变化 地址改变,生成新请求

      JSP 四个作用域

    • page 当前页 本页面
    • request 请求 本次请求
    • session 会话 当前会话 同一用户多次请求
    • application 应用程序 整个应用 当前项目范围

      Servlet 生命周期

    • 实例化:Tomcat启动时或Servlet第一次被访问
    • 初始化:该容器调用init()方法
    • 服务:如果请求Servlet,则容器调用Service()方法
    • 销毁:销毁实例前调用destroy()方法(Tomcat停止时)

      Servlet Request 接口

    • Object getAttribute(String name) 根据name属性名返回属性值(得到参数)
    • String getParameter(String name) 根据给定参数返回参数值,得到浏览器提交的参数
    • Void setAttribute(String name , Object object) 像浏览器传输Obje类型的数据,数据名字设定为name
    • request.getRequestDispatcher(“index.jsp”).forward(request, response) 跳转
    • request.setCharacterEncoding(“UTF-8”) 设定字符编码集
